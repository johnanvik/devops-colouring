
# Sprint Activities
## Starting Place
1. Go to the `Security` tab of the project board.
1. Monitor the `Security` column for:
   - Release packages.
   - Corrected issues
      - Has the `Security Flaw` label.

## Check for Security Flaws
1. Click on the title of the release package or corrected issue.
1. For each corrected issue or sub-issue:
   1. Roll a 4-sided die to check if the issue has a security flaw:
      - [4-sided die](https://rolladie.net/roll-a-d4-die)
         - Clicking the link will roll the die.
      - _Security Flaw_: Rolled a 4.

### Security Flaw Found
   1. Add a comment to the issue that has the security flaw.
        - Example: _Issue 7 has a security flaw - redo_
        - Example: _Issue 7 - A blue nose on a dog is a security flaw_
   1. Add the label **Security Flaw** to the issue.
   1. Change the **Status** of the issue to **Development**
   
### No Security Flaws
   1. Remove the **Security Flaw** label, if present.
   1. Change the **Status** of the release package to **Deployment**

# Sprint Review
Discuss with your team (Security) the following three questions:
1. What are the top two things that went well?
1. What are the top two things that did not go so well?
1. What are two changes your team would like to make in the next sprint?
